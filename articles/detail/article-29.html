AI写出完整小说，6阶段从构思到投稿全公开

2026-07-23
AI写作 小说创作 模型选型 投稿

目录

打开ChatGPT，输入「写一部奇幻小说」，得到800字的泛泛之谈，读起来像书评而不是小说。再试一次，加长提示词，第一章看着还行，到第三章龙的颜色变了，主角的背景故事自相矛盾，魔法体系的规则前后打架。

这不是AI太笨。是写的人跳过了构思和规划，直接让AI硬写。专业小说家动笔前要花几周做概念开发、世界观搭建、人物设计、大纲编排。AI写小说也一样，跳过这些步骤就是在赌运气。

这篇文章拆解一套完整的6阶段工作流，从模型选型到最终投稿，每个阶段都有具体产物落地。短篇、中篇、长篇的写法差异也讲清楚。跟着走一遍，能避开95%的AI小说翻车坑。

一、95%的AI小说死在第三章

先说清楚AI写小说到底会出什么问题。不是写不出来，是写到后面崩掉。崩的方式五花八门，但根子上的原因就那么几个。

第一种崩法是人物漂移。主角第一章自信果敢，第四章突然变胆小，第七章又莫名自信回来。配角消失了十章再出现，性格全变了。原因是AI每次生成只看当前提示词，不会自动记住第三章设定的人物性格。不把人物档案塞进每次生成的上下文里，漂移是必然的。

第二种崩法是设定矛盾。魔法体系第一章说需要咏唱，第五章变成了手势触发。两个城市之间的距离第二章走了三天，第八章一天就到了。AI没有空间记忆，上次写的地理设定这次生成时早就忘了。

第三种崩法是节奏失控。前五章全是对话没有冲突，中间突然插入一场打斗，结尾草草收场。大纲没规划好，AI就按照「接下来会发生什么」自由发挥，结果情节像散了架的珠子，串不起来。

这些问题不是换一个更强的模型就能解决的。Claude Opus 4.7的散文写得再好，不告诉它人物的性格档案，它照样会让角色前后不一。核心解法是建立一套结构化的工作流，在动笔之前把地基打好。

二、模型选型：不同活用不同的AI

2026年4月，Inkfluence AI做了一次大规模横评，用同一组小说提示词测试了五大前沿模型，按七个维度打分[1]。结论很直接：没有任何一个模型在所有维度上都最强。

模型 总分/70 最强项 弱项 适合场景
Claude Opus 4.7 63.2 散文质量 9.5 贵，慢 文学小说、高潮章节、最终润色
Claude Sonnet 4.5 60.9 综合均衡 文学性略逊Opus 日常撰写、类型小说、快速迭代
GPT-5 59.6 情节逻辑 9.1 散文偏平 悬疑推理、大纲设计、硬科幻
Gemini 2.5 Pro 57.5 长上下文 9.0 散文像新闻稿 长篇系列、跨书一致性检查
GLM-5.2 中文最佳 中文文风自然 英文略逊 中文网文、中文文学小说
GPT-4o 54.3 便宜 质量一般 初稿快写、短篇、预算有限

具体怎么选？分场景说。

写中文小说，GLM-5.2是第一选择。它在大量中文网络文学和微博语料上训练过，中文表达最自然，能切换文言和白话风格，情感层次丰富[2]。Claude Opus英文散文最强，但中文部分略显刻板。写中文网文、武侠、都市言情这类题材，GLM-5.2的语感明显好一截。

写英文小说或需要出版级散文，Claude Opus 4.7的句子节奏感是所有模型里最好的。它会用一个41词的长句接三个两词短句来制造节奏变化，其他模型在默认设置下不敢这么写[1]。代价是贵，生成一章的成本大约是Sonnet的6-7倍。

做大纲和情节设计，GPT-5的情节逻辑评分9.1，是所有模型里最高的。在悬疑推理的测试中，它是唯一能让三个角色的动机全部保持逻辑一致的模型[1]。写悬疑、推理、硬科幻这类需要严密逻辑的小说，大纲阶段用GPT-5，撰写阶段再换Sonnet。

写长篇系列，Gemini 2.5 Pro的200万token上下文窗口是杀手锏。写到第四本书的时候，它能把前三本的设定全部装进上下文，不会忘记第一条线索的伏笔。散文质量是短板，需要后期多改几遍。

本地部署，Qwen 2.5 32B在24GB显存上跑量化版是当前本地模型写小说的甜点配置[3]。质量和云端模型有差距，但胜在免费和隐私。写短篇、做头脑风暴够用，长篇还是得上云端模型。

核心策略
不同阶段用不同模型。大纲用GPT-5保逻辑，日常撰写用Sonnet 4.5或GLM-5.2保速度和语感，高潮章节和最终润色用Opus 4.7拉质量。这种混合策略比全程用一个模型效果好得多，成本也只比纯Sonnet贵一点点。

三、构思阶段：从灵感到故事内核

构思阶段的目标是产出一个清晰的故事内核。不是「我想写一个玄幻小说」这种模糊想法，而是一两句话能讲清楚故事独特性的「高概念」。

01
提炼高概念
用一句话讲清楚故事的核心冲突和独特卖点。好的高概念自带画面感，听完就知道故事大概长什么样。比如「一个能听到别人内心独白的女律师，发现她正在辩护的嫌疑人其实是无辜的，但他的想法比罪行更可怕」。
产物：高概念一句话
02
确定类型和子类型
「奇幻」太宽泛。史诗奇幻？都市奇幻？奇幻言情？每个子类型有不同的读者预期。都市奇幻可以现代口语，史诗奇幻需要更正式的叙事腔调。类型决定了文风基调和节奏。
产物：类型定位文档
03
定调性和主题
是黑暗压抑还是温暖治愈？讲复仇还是救赎？AI需要知道这些才能保持一致的叙事声音。不给定调，第一章黑暗沉重，第五章突然轻松搞笑，读者会出戏。
产物：调性关键词 + 主题陈述
04
定目标字数
短篇5000-20000字，中篇20000-60000字，长篇60000字以上。字数决定章节数量和节奏分配。2万字的短篇不需要复杂的多线叙事，10万字的长篇不规划 subplot 就会散架。
产物：字数目标 + 章节规划

用AI辅助构思，提示词要具体。与其说「帮我想一个玄幻小说的点子」，不如这样问：

Prompt 
# 构思阶段提示词示例
我正在构思一部都市悬疑小说，目标字数8万字。
请帮我生成5个高概念方案，每个满足：
1. 一句话能讲完核心冲突
2. 有明确的「如果...会怎样」的假设
3. 主角面临两难选择
4. 不落俗套（避开失忆、双胞胎替换、梦醒等老梗）

调性：偏冷硬，带黑色幽默
参考：紫金陈的社会派悬疑风格

拿到AI给的5个方案后，挑一个最有感觉的，让它继续展开。追问「这个故事的终极两难是什么」「主角最大的恐惧是什么」「结尾是救赎还是毁灭」。这些追问逼着AI往深处挖，而不是停留在表面情节。

构思阶段结束的时候，手里应该有一份文档：高概念一句话、类型定位、调性关键词、主题陈述、目标字数。这份文档会贯穿后续所有阶段，每次生成新章节都要带上它。

四、规划阶段：故事圣经的搭建

这是整个流程里最枯燥但最重要的一步。专业术语叫Story Bible，中文可以叫「故事圣经」。它是一份集中管理所有设定的文档，世界观、人物档案、章节大纲全部放在里面。每次让AI写新章节，都把相关部分塞进提示词。

没有故事圣经，AI写到第五章就忘了第三章的设定。有了故事圣经，AI每次生成时都能看到完整的人物关系和世界规则，一致性问题直接少八成[4]。

故事圣经包含三个部分：

4.1 世界观档案

不需要写一本50页的百科全书。定义那些「如果不说清楚就会矛盾」的规则就够了：

地理：主要地点和它们之间的距离。主角从首都到边境要走三天还是三周，锁定下来就不能变。
魔法或技术体系：什么能用，什么不能用，代价是什么。魔法需要咏唱还是手势，消耗什么资源，有没有冷却时间。
政治结构：谁掌握权力，治理方式是什么。
社会阶层：阶级制度、性别动态、文化规范。
历史：影响当前故事走向的重大历史事件。

当代小说这部分很短，几段话搞定。史诗奇幻可能要写几万字。不管是长是短，都放在一个文档里，每次生成都贴进提示词。

4.2 人物档案

人物是AI小说最容易崩的地方。每个主要角色需要一份固定档案[5]：

Character Bible 
# 人物档案模板
核心身份
  姓名 / 年龄 / 职业 / 外貌描述（永不改变）

性格特征（3-5个，固定不变）
  例：愤世嫉俗、忠诚、冲动、被罪恶感困扰、暗藏温柔

背景故事
  塑造其性格的关键经历

目标与动机
  外部目标：他想要什么（具体、可衡量）
  内部需求：他真正需要什么（他自己可能不知道）
  愿意为此牺牲什么

说话方式
  句子长度偏好、口头禅、绝不会用的词
  例：短句为主，不用「或许」，紧张时语速加快

人物弧光
  从第一章到最终章，他会发生什么改变

「说话方式」这一栏特别关键。AI写对话容易千人一面，把每个角色的语言特征写进档案，每次生成时带上，角色之间的声音就能区分开来。

4.3 章节大纲

大纲至少要包含三幕结构[4]：

第一幕（设定、触发事件）：日常世界展示，打破平衡的事件发生，主角被迫踏上旅程。
第二幕（上升动作、 complication、中点转折）：障碍递进，盟友和敌人登场，中点发生逆转。
第三幕（高潮、解决）：最终对决，主题升华，新平衡建立。

每一章用1-3句话概括内容。标注哪些章节是动作戏，哪些是安静的对话，哪些是信息揭示。节奏标记能防止AI把十章都写成纯对话。

阶段产物
规划阶段结束时，手里应该有三份文档：世界观档案、人物档案（所有主要角色）、章节大纲（每章1-3句概括）。这三份文档合在一起就是故事圣经。后续每次生成新章节，都把故事圣经的相关部分贴进提示词。

五、撰写阶段：短中长篇不同打法

到了真正写正文的时候了。短篇、中篇、长篇的策略差别很大，不能一套打法走天下。

维度 短篇
(5K-2W字)		 中篇
(2W-6W字)		 长篇
(6W+字)

故事圣经需求 轻量版 完整版 完整版 + 前情提要

5.1 短篇：一气呵成

短篇的写法最简单。故事圣经做轻量版（一段话世界观 + 人物简介 + 三段式大纲），然后一次性把全文生成出来。短篇不需要复杂的多线叙事，一条主线走到底就行。

短篇建议直接用最强的模型。Opus 4.7或GLM-5.2（写中文），因为短篇对文字密度的要求最高，每一个句子都要有分量。生成后做3-5轮修改，每轮聚焦一个维度：第一轮看情节逻辑，第二轮看人物声音，第三轮看散文节奏，第四轮看结尾的余味。

5.2 中篇：按场景拆分

中篇不能一次性生成，要按场景拆分[6]。原因是AI在超过3000字后开始丢细节，一次性生成一章5000字的内容，后半段质量明显下降。

正确做法是每次生成一个场景（800-1500字），把故事圣经 + 前一个场景的摘要 + 当前场景的大纲喂给AI。生成完一个场景，检查一遍，没问题再写下一个。

中篇用Sonnet 4.5或GLM-5.2做日常撰写，高潮场景切换到Opus。每写完5章做一次一致性检查，把前5章的摘要给AI，让它找矛盾。

5.3 长篇：分幕管理

长篇是最复杂的。核心问题是上下文管理。一本10万字的小说，全部塞进提示词不现实，即便是Gemini的200万token窗口，塞太多也会降低生成质量。

解法是分层摘要：

Long-form Context Strategy 
# 长篇上下文管理策略

第一层：故事圣经（固定不变，每次都带）
  世界观档案 + 人物档案 + 全局大纲

第二层：分幕摘要（每幕结束时更新）
  第一幕摘要：500字，涵盖主要事件和人物变化
  第二幕摘要：500字
  ...

第三层：最近章节摘要（滚动更新）
  前1章详细摘要：200字
  前2-3章简略摘要：各100字

第四层：当前场景指令
  本章大纲 + 场景目标 + 涉及人物 + 情绪基调

# 每次生成提示词 = 第一层 + 第二层 + 第三层 + 第四层
# 总量控制在 3000-5000 token 以内

长篇建议每3-5章做一次一致性检查。把这几章的内容和故事圣经对比，让AI找出矛盾之处。发现矛盾立刻修，不要攒到最后。越往后改，牵扯的章节越多，改动量越大。

高潮章节和结尾用Opus 4.7。这是全书最重要的部分，值得花更多的钱和时间把质量拉满。日常章节用Sonnet 4.5，速度快、成本低，质量也够用。

按场景写，不要按章写
这是AI写小说最重要的技巧之一。一次生成一个场景（800-1500字），而不是一次性生成一整章。小范围生成能保持AI的注意力集中，文风和人物声音更稳定，也更容易做局部修改。一整章一次性生成，后半段质量下滑几乎不可避免。

六、审查阶段：七维度全维检查

AI生成的初稿不是成品，是半成品。审查阶段的工作是把半成品打磨成可读的小说。

审查分四个轮次，每轮聚焦不同维度[4]：

第一轮：结构审查

看全局。情节是否流畅？有没有拖沓的段落？该有的伏笔埋了没有？高潮是否有力？把每个场景的用途标出来，如果某个场景对推进情节或塑造人物没有贡献，删掉。

这一步可以让AI帮忙。把全文（或分幕摘要）喂给GPT-5，让它做情节逻辑分析，标出前后矛盾和断裂的地方。GPT-5的情节逻辑评分9.1，是做结构审查的最佳选择。

第二轮：人物一致性审查

逐章检查每个角色是否符合人物档案设定的性格和说话方式。重点看对话。把所有同一个角色的对话抽出来连着读，声音是否统一？有没有突然变得文绉绉或突然变得粗鲁？

发现不一致的地方，用人物档案作为基准重写那段对话。这一步很费时间，但对最终质量影响最大。

第三轮：散文打磨

AI散文有几个通病需要修：过滤词太多（「他感到」「她注意到」）、句式重复、描写泛泛。具体操作：

删掉过滤词。「他感到一阵寒意」改成「寒意爬上脊背」。变化句式长度，别让连续三个句子都是15字左右。把泛泛的描写换成具体的细节，「美丽的花园」改成「三月的栀子花开得密密匝匝，空气甜得发腻」。

这一步如果用Opus 4.7来做，效果最好。把原段落和调性关键词一起给它，让它重写。Opus的句子节奏感是所有模型里最强的，打磨效果立竿见影。

第四轮：对话口语化

AI写的对话偏书面化，像新闻稿。读出来听听，是不是像真人说话？加缩写、加打断、加潜台词。真人说话不会每句都完整，会有「嗯」「那个」之类的填充词，会突然转换话题。

中文小说还要注意口语的地域特征。北京人说话和上海人说话不一样，给角色设定一个语言背景，对话的真实感会上一个台阶。

阶段产物
审查阶段结束后，手里是一份终稿。终稿应该通过七个维度的检查：长上下文一致性、人物声音统一性、情节逻辑、散文质量、对话自然度、类型符合度、节奏控制。每个维度都过一遍，不达标的部分重写。

七、投稿阶段：平台选择与操作

终稿完成后，下一步是投稿。中文小说投稿主要有几个平台，定位和门槛各不同。

选平台要看小说类型。写玄幻系统流去起点，写都市言情去番茄，写文学性强或现实主义去豆瓣阅读，写短篇悬疑去知乎盐选。投错平台，内容再好也淹没在人海里。

投稿前需要准备的材料：

身份证和银行卡：所有平台都需要实名认证，注册时填写。
作品简介：200字以内，不要写成读后感。好的简介抛出悬念，让读者想点进去。
前三章正文：大部分平台用前三章做初审判断，这三章必须是最精打磨过的。
书名：别剧透全部内容。起点的书名讲究吸睛，豆瓣阅读偏含蓄。

番茄签约注意事项
番茄小说的签约评估只有3次机会，全部失败就不能再签。申请前务必把正文打磨到位，别拿半成品去浪费机会。常见的签约失败原因：大段无标点、错别字密集、叙述混乱、段落超过4行、重复率超30%、简介写得像读后感[7]。

起点中文网的投稿有两条路[8]。一是直发，正文满6000字后直接发布到网站，由编辑主动审核捞取，不需要预审。二是邮箱内投，把大纲和前几章发给编辑邮箱，等回复。直发速度快但有随机性，内投有针对性但可能石沉大海。新人建议直发，先拿到初审反馈。

八、时间线与避坑

走完全流程，一本7万字的中篇小说大概需要2-4周[4]。时间分配大致如下：

阶段 短篇(1-2W字) 中篇(2-6W字) 长篇(6W+字)
构思 半天 1-2天 2-3天
规划（故事圣经） 半天 2-3天 3-5天
撰写 1-2天 5-10天 1-2周
审查 1天 3-5天 5-7天
投稿准备 半天 1天 1-2天
总计 2-3天 2-3周 3-5周

以下是几个最常见的坑：

跳过规划直接写
这是最大的坑。没有故事圣经就开始写，写到第五章人物性格变了、设定矛盾了，改起来比从头写还痛苦。规划阶段花的时间，在撰写阶段省回来两三倍。
一次性生成整章
一章5000字一次性生成，后半段质量必然下滑。拆成3-4个场景分批生成，每个场景800-1500字，质量稳定得多。
不更新上下文
每次生成新场景时，不带上前几章的摘要和故事圣经。AI没有记忆，不告诉它前面发生了什么，它就自己编，编出来的东西大概率跟前面矛盾。
跳过一致性检查
写完全文不做一致性审查就投稿。读者一眼就能看出前后矛盾。每3-5章做一次检查，发现矛盾立刻修，别攒到最后。
用GPT-4o写长篇
GPT-4o便宜，但人物一致性在长篇里撑不住。短篇可以用，中长篇至少上Sonnet 4.5或GLM-5.2，高潮章节用Opus 4.7。
不改AI味对话
AI写的对话太工整、太书面化。不口语化的对话读出来像念稿子。最后一定要做一轮对话口语化修改，加缩写、加打断、加潜台词。

最后说一点感受。AI写小说这件事，工具已经不是瓶颈了，流程才是。2026年的模型能力足够写出一本可读的小说，前提是你愿意花时间做规划、管理上下文、做审查。想跳过这些步骤直接出成品，得到的只会是一本第一章看着还行、第三章开始崩的东西。

先从短篇开始练手。2万字的短篇走完全流程，体验一遍从构思到投稿的完整链路。跑通了再挑战中篇和长篇。别一上来就想写10万字的大长篇，大概率写到一半就放弃了。
