# 酷炫个人主页网站实现计划

## Summary

构建一个单页个人主页网站，核心定位是"去AI味的工程师个人品牌页"。以深色终端美学为基底，通过粗野主义排版、单一荧光强调色、等宽字体介入、真实内容叙事，打造冷静、专注、略带叛逆的技术人格。页面包含：全出血 Hero（含叙事型轮播图）、个人介绍、工作经历时间线、开源项目展示、在线服务介绍、联系 CTA。

## Current State Analysis

目标目录 `d:\ai_work\smart-open.github.io` 已存在，当前包含一个 `p/` 子目录（内有 `g/` 游戏页面和 `s/` 某个工具站点），以及根级 `README.md`。本次计划在根目录新建主页文件，与现有 `p/` 目录共存，形成"主站 + 子项目"的站点结构。不改动现有文件。

## Proposed Changes

### 视觉论点

> "在精密的暗色代码海洋中，一抹锐利的光标在闪烁——这是人类工程师的痕迹。"

- **情绪**：冷静、专注、略带叛逆
- **材质**：磨砂黑金属、单色荧光、纸张纹理
- **能量**：低频率但高冲击，像深夜编译成功的终端提示

### 动画策略（4 个有意图的动画）

1. **Hero 轮播图切换 — "代码编译"感**
   - 触发：用户点击/键盘切换
   - 效果：当前 slide 向左滑出（带轻微模糊），新 slide 从右滑入，扫描线掠过
   - 技术：CSS transform + opacity + 伪元素扫描线
   - 叙事目的：像在不同项目窗口间切换

2. **Scroll-triggered 文字逐行浮现**
   - 触发：滚动进入视口
   - 效果：`translateY(40px)` + `opacity: 0` 过渡到正常，stagger 0.08s
   - 技术：GSAP ScrollTrigger
   - 叙事目的：阅读节奏感，引导视线深入

3. **竖向时间轴"绘制"与节点激活**
   - 触发：滚动进入工作经历 section
   - 效果：中央竖线从顶部向下以强调色"绘制"点亮；每个节点随滚动从灰色渐变为荧光色并触发脉冲光晕；内容块 stagger 浮现
   - 技术：GSAP ScrollTrigger 进度绑定竖线高度 + CSS animation 节点脉冲
   - 叙事目的：模拟代码从上到下的执行过程，技术生涯像程序一样逐步构建

4. **打字机效果（Hero 副标题）**
   - 触发：页面加载完成
   - 效果：逐个字符出现，光标闪烁
   - 技术：纯 JS setInterval，JetBrains Mono
   - 叙事目的：第一眼确立"这是人写的"

### 轮播图叙事设计

轮播图位于 Hero Section 内作为背景层，品牌名和文案浮于其上。

| Slide | 内容 | 叙事任务 |
|---|---|---|
| Slide 1 | 终端截图风格：项目启动命令 + 成功编译输出 | "我从命令行开始" |
| Slide 2 | 架构图/系统拓扑（手绘风格 SVG） | "我设计系统" |
| Slide 3 | 真实项目运行界面（数据仪表盘） | "我交付成果" |
| Slide 4 | GitHub 贡献图 / 开源社区互动 | "我回馈社区" |

交互：不自动播放（或 8 秒间隔+悬停暂停）、左右箭头、底部细线进度条、键盘切换、触摸滑动。

## Assumptions & Decisions

| 决策项 | 选择 | 理由 |
|---|---|---|
| 技术栈 | 纯 HTML + CSS + Vanilla JS，GSAP 仅用于 ScrollTrigger | 单页作品集无需组件化状态管理；零依赖可直接部署 GitHub Pages；完全控制 DOM，动画性能最优；避免"用 React 写静态页"的过度工程感，契合"去 AI 味"理念 |
| 部署 | GitHub Pages / Vercel / Cloudflare Pages | 纯静态，零构建步骤即可部署 |
| 是否使用构建工具 | 否（零构建） | 保持 raw 感，减少工具链带来的模板化气息 |
| 图片策略 | 初期使用占位图/生成极简抽象图 | 确保叙事结构先可用，后续替换为真实截图 |
| 响应式 | 必须支持 | 移动端单列、字体缩放、触摸手势 |
| 可访问性 | `prefers-reduced-motion` 支持 | 减少运动偏好用户获得降级体验 |

## 设计红线（必须遵守）

1. **无卡片布局** — 没有 `border-radius`、`box-shadow` 的浮起卡片
2. **无装饰性渐变** — 背景是纯黑色系，无蓝紫渐变
3. **无漂浮仪表盘/3D mockup** — 展示真实截图或极简 SVG
4. **无通用 SaaS 网格** — 不用三列等宽图标+标题+描述的重复模块
5. **无无叙事目的的轮播** — 每张 slide 必须有明确的叙事任务
6. **最多两种字体** — Inter + JetBrains Mono
7. **一种强调色** — 仅 `#00f0ff`，不混用其他亮色

## Verification Steps

1. **首屏检查**：打开页面，品牌名在首屏清晰可见，轮播图不遮挡文字，打字机效果正常播放
2. **导航检查**：点击锚点导航，平滑滚动到对应 section，无偏移或错位
3. **轮播检查**：左右箭头、键盘方向键、触摸滑动均可切换 slide，底部进度条同步更新
4. **滚动动画检查**：向下滚动，各 section 大标题有 stagger 浮现效果，竖向时间轴中央轴线从顶部向下以强调色绘制点亮，各节点依次激活脉冲光效
5. **响应式检查**：在移动端宽度（<768px）下，布局变为单列，文字可读，轮播图按钮可点击（>=44px）
6. **性能检查**：Lighthouse 性能评分 >90，无 layout shift，动画仅使用 transform/opacity
7. **可访问性检查**：开启系统"减少动态效果"，所有动画优雅降级为直接显示
